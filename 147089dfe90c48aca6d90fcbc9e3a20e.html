HTTP 1.0、HTTP 1.1、HTTP 2.0的区别

HTTP 计算机网络

HTTP 1.0

  1. 引入了HTTP头部
  2. 由于HTTP头部的引入，HTTP响应可以直接返回非HTML文件

HTTP 1.1

  1. 引入了持久化连接。（TCP连接默认不关闭，如果双方在一段时间之后仍没有发送数据，连接将被关闭）
  2. 管道机制。（允许在第一个请求的应答被完全发送之前就发送第二个请求，降低通信延迟。）

HTTP 2.0

  1. 二进制协议。不再可读。
  2. 多路复用。客户端可以同时发送多个请求，且顺序不用一一对应。
  3. 首部压缩。
  4. 服务端推送。（在客户端请求网页内部的静态资源，服务端主动推送）
