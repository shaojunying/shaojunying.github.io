连接docker时中文乱码问题

实践

问题描述

创建Container

今天使用如下命令创建了一个MySQL容器。

docker run -p 3306:3306 --name mysql --platform linux/amd64 -e MYSQL_ROOT_PASSWORD=root -v /Users/shaojunying/WorkSpace/seckill/src/main/sql/:/usr/root/sql mysql:8.0

（上述命令中的 -v /Users/shaojunying/WorkSpace/seckill/src/main/sql/:/usr/root/sql 在容器上挂载了一个目录。）

进入Container并执行SQL脚本（遇到中文乱码）

之后在新的窗口执行命令进入MySQL容器的命令行

docker exec -it mysql /bin/bash

之后执行命令进入MySQL

mysql -u root -p

之后执行如下命令初始化数据库

source /usr/root/sql/schema.sql

以上命令均执行正常。下一步准备使用如下命令向表中插入数据，但是发现不能插入中文。

insert into product(name, stock, start_time, end_time) values("100元秒杀MacBook Air", 100, "2021.10.05 00:00:00", "2021.10.05 10:00:00");

另外在intelij IDEA中连接数据库查询表信息，发现表每个列的comment都是乱码（comment都是用中文写的。）

解决办法

使用如下命令重新进入Container。

docker exec -it mysql env LANG=C.UTF-8 /bin/bash

之后重新连接MySQL并重新执行SQL脚本，再次执行包含中文的插入语句，发现中文可以被正确地插入。

之后又使用intelij IDEA，发现中文的comment也可以正常显示。
