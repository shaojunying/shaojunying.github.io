LeetCode 851 喧闹和富有

Posted on Wed, Dec 15, 2021 算法 数据结构 LeetCode 图 拓扑排序

图+拓扑排序

拓扑排序

定义

拓扑排序一般用来在一个依赖关系集当中，确定事务发生的顺序。保证如果B依赖A，那么A一定先于B发生。

实现

拓扑排序一般是用图来表示，将事务表示为图中的各个节点，事务之间的依赖关系对应于图中的边。（事务B依赖事务A 将对应与 图中一条A→B的边。）

  1. 之后每次从图中选择一个入度为0的节点，删除所有以该节点为起点的边。
  2. 更新各个节点的入度，重复步骤一。

（如果图中不存在环，那么重复上面两步可以取出所有节点。节点取出的顺序就是一个拓扑排序）

本题的思路

本题中，我们需要得到ans数组：对于任一person x，ans[x] = y表示的是在所有拥有的钱肯定不少于person x的人中，person y是最安静的人。

我们先考虑第一个条件：在所有拥有的钱肯定不少于person x：我们将a比b有钱看作b依赖于a，得到一个拓扑排序。在得到拓扑序的过程中，删除边的同时，更新每个节点的ans值，最终即可得到每个节点的结果。

代码

class Solution {
    public int[] loudAndRich(int[][] richer, int[] quiet) {
        // 根据richer建图 [i, j]对应i指向j的一条边
        int n = quiet.length;
        int[][] graph = convertToGraph(richer, n);
        int[] indegrees = getIndegree(graph, n);
        // 入队所有入度为0的节点
        Queue<Integer> queue = new ArrayDeque<>();
        for (int i = 0; i < indegrees.length; i ++) {
            if (indegrees[i] == 0) {
                queue.add(i);
            }
        }
        // 为每个节点赋默认值
        int[] res = new int[n];
        for (int i = 0; i < res.length; i ++) {
            res[i] = i;
        }
        while (!queue.isEmpty()) {
            int element = queue.remove();
            for (int j = 0; j < graph.length; j ++) {
                if (graph[element][j] == 1) {
                    // 有element指向j的边

                    // 尝试更新 钱数大于等于j的钱数 且安静值最小的元素
                    // 前一点肯定满足 只需要考虑后面一点
                    if (quiet[res[element]] < quiet[res[j]]) {
                        res[j] = res[element];
                    }
                    
                    // 更新j的入度信息
                    indegrees[j] --;
                    if (indegrees[j] == 0) {
                        queue.add(j);
                    }
                }
            }
        }
        return res;
    }

    private int[][] convertToGraph(int[][] matrix, int n) {
        int[][] graph = new int[n][n];
        for (int[] pair : matrix) {
            int node1 = pair[0];
            int node2 = pair[1];
            graph[node1][node2] = 1;
        }
        return graph;
    }

    private int[] getIndegree(int[][] graph, int n) {
        int[] indegrees = new int[n];
        for (int i = 0; i < graph.length; i ++) {
            for (int j = 0; j < graph.length; j ++) {
                // 更新节点的入度
                if (graph[i][j] == 1) {
                    indegrees[j] ++;
                }
            }
        }
        return indegrees;
    }

}

时间复杂度

O(m+n)O(m + n)O(m+n)

深度优先遍历

思路

同样构建图，但是与上一步相反，这里对于a比b富有，构造一条b到a的边。

这样从任一节点开始执行深度优先遍历，将可以遍历到所有比b富有的人，找到这些人中安静度最小的人即为答案。

时间复杂度

O(n2)O(n^2)O(n2) 可以缓存已计算得到ans的节点，优化为O(N+M)O(N + M)O(N+M)
