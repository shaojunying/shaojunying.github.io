Java中NIO的三大组件

Java NIO的三大组件分别是Buffer、Channel、Selector。

Buffer

一个Buffer本质上是一块内存区域，我们可以将数据写入这块内存中；或从这块内存区域中读取数据。

Buffer的核心实现是ByteBuffer，其余实现还包括ByteBuffer的一些包装类：IntBuffer、DoubleBuffer。在使用时，我们可以将Buffer理解为一个数组。ByteBuffer看作byte[], IntBuffer看作int[]……

Channel（通道）

所有的NIO操作始于通道，通道是数据来源或数据写入的目的地。java.nio包中的实现包含以下四种：

  • FileChannel：文件通道，用于文件的读写
  • DatagramChannel：用于UDP连接的接收和发送
  • SocketChannel：TCP的连接通道
  • ServerSocketChannel：TCP的服务端，用于坚挺某个端口进来的请求。

读操作的时候将Channel中的数据填充到Buffer中；写操作时将Buffer中的数据写入到Channel中。

FileChannel 是不支持非阻塞的，我们说NIO一般指的是网络IO。

Selector

Selector建立在非阻塞的基础上，用于实现一个线程管理多个Channel。我们常听说的多路复用在Java中指的就是Selector。

阻塞模式IO（BIO）

阻塞IO的瓶颈

  • 为每个连接开一个线程代价很高。每个线程都需要一部分内存，如果连接数是几万几十万时，内存将会被迅速消耗；此外线程切换的开销非常大。
  • 阻塞操作会导致CPU的空转，降低CPU的使用效率。

代码实现

Server

package bio;

import java.io.IOException;
import java.net.InetSocketAddress;
import java.nio.ByteBuffer;
import java.nio.channels.ServerSocketChannel;
import java.nio.channels.SocketChannel;
import java.nio.charset.StandardCharsets;
import java.util.Arrays;

public class Server {

    public static void main(String[] args) throws IOException {
        ServerSocketChannel serverSocketChannel = ServerSocketChannel.open();
        serverSocketChannel.bind(new InetSocketAddress(8080));
        while (true) {
            SocketChannel socketChannel = serverSocketChannel.accept();
            new WorkerThread(socketChannel).start();
        }
    }

    public static class WorkerThread extends Thread {
        private final SocketChannel socketChannel;

        public WorkerThread(SocketChannel socketChannel) {
            this.socketChannel = socketChannel;
        }

        @Override
        public void run() {
            ByteBuffer buffer = ByteBuffer.allocate(1024);
            try {
                // 将请求数据读入 Buffer 中
                int num;
                while ((num = socketChannel.read(buffer)) > 0) {
                    // 读取 Buffer 内容之前先 flip 一下
                    buffer.flip();

                    // 提取 Buffer 中的数据
                    byte[] bytes = new byte[num];
                    buffer.get(bytes);

                    String re = new String(bytes, StandardCharsets.UTF_8);
                    System.out.println("收到请求：" + re);

                    // 回应客户端
                    ByteBuffer writeBuffer = ByteBuffer.wrap(("我已经收到你的请求，你的请求内容是：" + re).getBytes());
                    socketChannel.write(writeBuffer);

                    buffer.clear();
                }
                socketChannel.close();
            } catch (IOException e) {
                e.printStackTrace();
            }
        }
    }

}

Client

package bio;

import java.io.IOException;
import java.net.InetSocketAddress;
import java.nio.ByteBuffer;
import java.nio.CharBuffer;
import java.nio.channels.SocketChannel;
import java.util.Arrays;

/**
 * 客户端每秒向服务端发送一个Hello World
 */
public class Client {

    public static void main(String[] args) throws IOException {
        SocketChannel socketChannel = SocketChannel.open();
        socketChannel.connect(new InetSocketAddress("127.0.0.1", 8080));
        // 发送请求
        ByteBuffer buffer = ByteBuffer.wrap("1234567890".getBytes());
        socketChannel.write(buffer);

        // 读取响应
        ByteBuffer readBuffer = ByteBuffer.allocate(1024);
        int num;
        if ((num = socketChannel.read(readBuffer)) > 0) {
            readBuffer.flip();

            byte[] re = new byte[num];
            readBuffer.get(re);

            String result = new String(re, "UTF-8");
            System.out.println("返回值: " + result);
        }
        socketChannel.close();
    }

}

非阻塞IO（NIO）

非阻塞IO的核心在于使用一个Selector来管理多个通道（Channel）。

  • 将各个通道注册到Selector上，指定要监听的事件。
  • 只使用一个线程来轮询这个Selector，看上面是否有通道准备好。
    • 只有当通道准备好之后才真正地开始读写，避免为每个通道创建一个线程，提升效率。

Selector在不同操作系统下的实现

NIO中的Selector是对底层操作系统实现的一个抽象，不同操作系统中的实现如下：

  • Linux下的实现
    • Select。在操作系统层面轮询，每次最多轮询1024个socket。
    • Poll。Select的替代者，最大的区别就是poll不再限制socket的最大数量。
    • Select和Poll都有一个共同的问题。它们只会告诉你有几个通道准备好了，但不会告诉你具体是哪几个通道，需要自己进行一次扫描。时间复杂度为O(N)
    • Epoll。可以直接返回具体的准备好的通道。时间复杂度为O(1).
  • Windows下的实现
    • Select。Windows下的非阻塞IO使用Select。Windows中IOCP提供的异步IO是十分强大的。

JVM就是一个屏蔽底层实现的平台，我们直接面向Selector编程就可以了。

代码实现

public class SelectorServer {

    public static void main(String[] args) throws IOException {
        Selector selector = Selector.open();

        ServerSocketChannel server = ServerSocketChannel.open();
        server.socket().bind(new InetSocketAddress(8080));

        // 将其注册到 Selector 中，监听 OP_ACCEPT 事件
        server.configureBlocking(false);
        server.register(selector, SelectionKey.OP_ACCEPT);

        while (true) {
            int readyChannels = selector.select();
            if (readyChannels == 0) {
                continue;
            }
            Set<SelectionKey> readyKeys = selector.selectedKeys();
            // 遍历
            Iterator<SelectionKey> iterator = readyKeys.iterator();
            while (iterator.hasNext()) {
                SelectionKey key = iterator.next();
                iterator.remove();

                if (key.isAcceptable()) {
                    // 有已经接受的新的到服务端的连接
                    SocketChannel socketChannel = server.accept();

                    // 有新的连接并不代表这个通道就有数据，
                    // 这里将这个新的 SocketChannel 注册到 Selector，监听 OP_READ 事件，等待数据
                    socketChannel.configureBlocking(false);
                    socketChannel.register(selector, SelectionKey.OP_READ);
                } else if (key.isReadable()) {
                    // 有数据可读
                    // 上面一个 if 分支中注册了监听 OP_READ 事件的 SocketChannel
                    SocketChannel socketChannel = (SocketChannel) key.channel();
                    ByteBuffer readBuffer = ByteBuffer.allocate(1024);
                    int num = socketChannel.read(readBuffer);
                    if (num > 0) {
                        // 处理进来的数据...
                        System.out.println("收到数据：" + new String(readBuffer.array()).trim());
                        ByteBuffer buffer = ByteBuffer.wrap("返回给客户端的数据...".getBytes());
                        socketChannel.write(buffer);
                    } else if (num == -1) {
                        // -1 代表连接已经关闭
                        socketChannel.close();
                    }
                }
            }
        }
    }
}

NIO.2 （AIO）异步IO

我们会在很多场景使用异步。通常我们会使用一个线程池执行异步任务。提交任务的线程将任务提交到线程池之后立马返回，不必等到任务真正完成。如果想知道任务的执行结果，通常是传递一个回调函数，任务结束后去调用这个函数。

Java中的异步IO也是一样的，都是由一个线程池来负责执行任务，然后使用回调或自己去查询结果。

底层实现

  • Windows下使用IOCP来实现，操作系统管理线程池，性能非常好。
  • Linux下也有异步IO系统实现，但是限制较多且性能一般。因此JDK使用自建线程池的方式实现。

Java异步IO的两种使用方式

返回Future实例

Future提供如下的方法用于获取任务的执行情况：

  • future.isDone();

    判断操作是否已经完成，包括了正常完成、异常抛出、取消

  • future.cancel(true);

    取消操作，方式是中断。参数 true 说的是，即使这个任务正在执行，也会进行中断。

  • future.isCancelled();

    是否被取消，只有在任务正常结束之前被取消，这个方法才会返回 true

  • future.get();

    这是我们的老朋友，获取执行结果，阻塞。

  • future.get(10, TimeUnit.SECONDS);

    如果上面的 get() 方法的阻塞你不满意，那就设置个超时时间。

提供回调函数

java.nio.channels.CompletionHandler 接口定义：

public interfaceCompletionHandler<V,A> {
    void completed(V result, A attachment);
    void failed(Throwable exc, A attachment);
}
注意，参数上有个 attachment，虽然不常用，我们可以在各个支持的方法中传递这个参数值
AsynchronousServerSocketChannel listener = AsynchronousServerSocketChannel.open().bind(null);

// accept 方法的第一个参数可以传递 attachment
listener.accept(attachment, new CompletionHandler<AsynchronousSocketChannel, Object>() {
    public voidcompleted(
      AsynchronousSocketChannel client, Object attachment) {
          //
      }
    public voidfailed(Throwable exc, Object attachment) {
          //
      }
});

Java异步IO的使用

Java异步IO的使用中我们需要关注三个类，分别AsynchronousSocketChannel ，AsynchronousServerSocketChannel 和 AsynchronousFileChannel ，只不过是在之前介绍的 FileChannel、SocketChannel 和 ServerSocketChannel 的类名上加了个前缀 Asynchronous 。

AsynchronousFileChannel

文件IO在所有操作系统中都不支持非阻塞模式，但是我们可以对文件IO采用异步IO方式来提高性能。
