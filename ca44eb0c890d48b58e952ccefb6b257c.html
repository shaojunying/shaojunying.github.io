Python的GIL和多线程的关系

Python 多线程

定义

Python中的GIL全称为：全局解释器锁。只有持有GIL锁的线程才能在CPU上运行，从而保证了同一时间只有一个Python线程正在运行。这种方式可以保证线程安全。CPU执行每个线程的过程如下：

// 获取GIL锁
// 执行代码
// 释放GIL锁

每执行固定时间的字节码，或者遇到IO操作，Python线程将释放GIL锁，从而其他线程可以获取GIL锁。

影响

  1. 对IO密集型应用影响较小（但也会有影响）。这种应用会频繁执行IO操作并开始等待，等待之前会释放GIL锁。进而保证了IO等待时CPU可以执行其他线程。
  2. 对CPU密集型应用影响较大。由于GIL锁的存在，会导致多核多线程的程序退化成多核单线程，而CPU密集型应用应该同时在多核上执行，这样效率才能最大。Python的GIL锁将严重限制这种应用的执行效率。

解决方案

  1. 使用multiprocessing提供的多进程。通过进程池来执行CPU密集型任务可以利用多核CPU的优势。（IO密集型应用也可以尝试使用多进程，对于之前的抢票脚本，只使用线程池每秒可以发送89个请求，如果使用多进程+线程池，每秒可以发送132个请求。）
